# Aliolo Web & Synchronization Guide

This guide outlines the steps to deploy Aliolo as a web application and enable cloud synchronization using Supabase.

## 1. Supabase Setup

1.  Create a project at [supabase.com](https://supabase.com).
2.  **Authentication:** Enable Email/Password provider.
3.  **Database:** Create the following tables in the SQL Editor:

```sql
-- Users Table (linked to auth.users)
create table public.users (
  id uuid references auth.users not null primary key,
  username text unique,
  email text unique,
  role text check (role in ('superuser', 'manager', 'user')),
  total_xp integer default 0,
  current_streak integer default 0,
  max_streak integer default 0,
  theme_mode text default 'system',
  ui_language text default 'en',
  updated_at timestamp with time zone default now()
);

-- Cards Table
create table public.cards (
  file_id text primary key,
  pillar text,
  subject text,
  level integer,
  prompts text[],
  answers text[],
  video_url text,
  is_deleted boolean default false,
  updated_at timestamp with time zone default now()
);

-- Progress Table
create table public.progress (
  id bigint generated by default as identity primary key,
  user_id uuid references auth.users not null,
  card_file_id text not null,
  subject text not null,
  correct_count integer default 0,
  ease_factor float default 2.5,
  interval integer default 0,
  repetition_count integer default 0,
  next_review timestamp with time zone,
  updated_at timestamp with time zone default now()
);
```

4.  **Storage:** Create a bucket named `card-assets` and set it to "Public".

## 2. Connect the App

Update the credentials in `aliolo/lib/data/services/auth_service.dart`:

```dart
await Supabase.initialize(
  url: 'https://YOUR_PROJECT_ID.supabase.co',
  anonKey: 'YOUR_ANON_KEY',
);
```

## 3. Build for Web

Run the following command in your terminal:

```bash
cd aliolo
flutter build web --release
```

The output will be in `build/web/`. You can host this on **Vercel**, **Netlify**, or **Firebase Hosting**.

## 4. Synchronization Logic (Next Steps)

The app is now "Sync-Ready":
- `CardModel`, `UserModel`, and `ProgressRecord` now have `updatedAt` and `isDeleted` fields.
- `AuthService` is ready to handle Supabase sessions.
- You can now implement a `SyncService` that pulls/pushes data based on the `updatedAt` timestamps.

## 5. Offline Support

The app continues to use **Isar** for local storage. 
- **Offline:** Users can study and create content; it's saved locally with a new `updatedAt` timestamp.
- **Online:** The app detects a connection and syncs the latest `updatedAt` records with Supabase.
